# AI 设置点击厂商白屏修复 — 真机测试指南（2026-07-08）

## 一、问题现象
在「AI 接入设置」页（`/settings/ai`），点击「AI 服务商」下拉切换某个厂商时，页面跳成**白屏**，控制台报错：
```
livewire.js?id=...:4824 Uncaught Could not find Livewire component in DOM tree
```
后果：无法保存 AI 设置、无法接入模型，进而概念图谱等依赖 AI 的能力也测不了。

## 二、根因（第一性原理）
`settings.blade.php`（页面视图）已经用 `<x-app-layout>` 包裹了 `<livewire:ai-settings />`，
而 `resources/views/livewire/ai-settings.blade.php`（Volt 组件）**自己又套了一层 `<x-app-layout>`**。

`x-app-layout` 渲染的是**完整 HTML 文档**（`<!DOCTYPE html><html><head>…<body>`）。
于是整页被嵌了两层 `<html>`：外层是页面，内层（Livewire 组件根）是另一个 `<html>`，
且内层 `<html>` 处于 `<body>` 内部的非法 DOM 位置。

- 初次整页加载：浏览器勉强容错解析，能显示；
- 一旦点厂商触发 `wire:change="applyPreset"` 做**局部重渲染**：Livewire 用 morphdom 从组件根开始 diff，
  但根是非法位置的嵌套 `<html>`，morphdom 找不到合法根节点 → 抛 `Could not find Livewire component in DOM tree` → 重渲染中断 → 内容清空 = 白屏。

对照：其它嵌入式 Volt 组件（`dashboard`/`flashcards`/`stats`/`reader`/`profile` 表单等）都只输出单个根 `<div>`、不带自己的布局，只有 `ai-settings` 违规。

## 三、修复
去掉 `ai-settings.blade.php` 里多余的 `<x-app-layout>` 包裹，让它只输出一个根 `<div class="space-y-6">`
（与 `dashboard` 等嵌入式组件的范式一致）。布局由外层 `settings.blade.php` 提供，不重复。

文件：`resources/views/livewire/ai-settings.blade.php`
- 开头：`<x-app-layout>\n<div class="space-y-6">` → `<div class="space-y-6">`
- 结尾：`</div>\n</x-app-layout>` → `</div>`

> 因之前跑过 `view:cache` 会把 Volt 组件编译缓存，修复后已执行 `view:clear` + `view:cache` 重新编译，并重启 `artisan serve`。

## 四、自测结果（开发侧已通过）
- 真 HTTP 抓取 `/settings/ai`（带登录态）：
  - `<html` 出现次数 = **1**（修复前 2，确认不再嵌套文档）
  - `<body` 出现次数 = **1**
  - `ai-settings` 组件根为合法 `<div wire:id="…" wire:snapshot=…>`，名字 `ai-settings`
  - 页面无 `app-layout` 残留
  - 初始状态正确：`provider=openai, format=openai, baseUrl=https://api.openai.com/v1, model=gpt-4o-mini`

## 五、真机测试步骤（你来验）
前置：开发服务器已在 `http://127.0.0.1:8123` 用修复后代码重启。

1. 浏览器打开 `http://127.0.0.1:8123/enter`（自动登录本机账户），进入书架。
2. 点右上/侧边「⚙️ AI 设置」（或直访 `http://127.0.0.1:8123/settings/ai`）。
3. 在「AI 服务商」下拉里**逐个切换**不同厂商（OpenAI / Anthropic / Gemini / 自定义 等）：
   - **预期**：切换时页面不白屏；选中「自定义」会多出一个「API 协议」下拉；选不同厂商会自动填入对应的 Base URL / 模型名。
   - 不再出现 `Could not find Livewire component in DOM tree` 控制台报错。
4. 填一个 API Key（或留空走离线演示），点「保存设置」/「保存并测试连接」：
   - **预期**：提示「已保存 ✓」；点测试连接显示 ✅/⚠️ 连通性结果，不再白屏。
