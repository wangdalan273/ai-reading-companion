# 文本笔记 · 布局方案设计说明

> 配套原型：`文本笔记布局-多方案对比.html`（顶栏一键切换 6 种布局，共用同一设计系统，可直接对比）
> 适用产品：AI 伴读 · 文本笔记模块

---

## 0. 统一设计系统（保证风格一致、便于横向对比）

| Token | 值 | 说明 |
|---|---|---|
| `--paper` | `#f4f1ea` | 页面底色：暖纸 |
| `--surface` | `#ffffff` | 卡片/表面 |
| `--ink` | `#222631` | 主文字（与纸对比 > 4.5:1，WCAG AA）|
| `--accent` | `#c0563b` | 主强调：陶土橙（阅读陪伴感，避开通用蓝/紫渐变）|
| `--gold` | `#b08948` | 次级强调：书页金 |
| 圆角 | 14 / 10 / 7 | 卡片 / 小组件 / 微型 |
| 阴影 | sm / md / lg | 三级，hover 升级 |
| 间距 | 基准 4px，gap 18px，卡内距 14px | 8pt 体系 |
| 字号 | 15 / 13 / 11.5 | 标题 / 正文 / 弱 |
| 触控 | ≥ 44px | 所有可点元素 |

图标：统一 inline SVG（线性 1.8pt，currentColor），按笔记类型取色（doc/quote/list/people/scale/chart/bookmark/chat/bulb/scissors）。

---

## 1. 卡片网格（Card Grid）

- **图标排版与布局**：类型图标置于卡片左上角 34px 圆角徽标（按类型取色）；操作图标「置顶 / 更多」置于右上角 17px 灰色，悬停加深。头部 space-between 两端对齐。
- **信息层级与视觉重点**：标题（衬线 15px/600）第一视觉；摘要（13px/灰）限 3 行截断；标签 chips 第二层级；底部日期·字数最弱。置顶卡片左侧 3px 陶土橙边。
- **交互元素摆放**：整卡可点；右上操作悬停浮出；hover 整卡上浮 3px + 阴影加深。
- **多屏适配**：移动端单列 → ≥640 两列 → ≥1024 三列 → ≥1280 四列（`auto-fill minmax(258px)`）。

## 2. 瀑布流（Masonry · CSS columns）

- **图标排版与布局**：与卡片网格完全一致（左上徽标 + 右上操作），跨方案视觉统一。
- **信息层级与视觉重点**：层级相同，但内容长度不一形成错落节奏；重点笔记可加 64px 渐变封面 + 字母锚点。
- **交互元素摆放**：同卡片网格；`break-inside:avoid` 防截断。
- **多屏适配**：`column-count` 1→2→3→4，间距固定 18px。注：CSS columns 为纵向填充，需严格时间序时改 JS masonry。

## 3. 列表（List）

- **图标排版与布局**：左侧 56px 方形缩略图（封面渐变 + 类型图标叠印）垂直居中；中信息列；右操作列（日期 + 置顶/更多）右对齐。三栏栅格。
- **信息层级与视觉重点**：标题 14px 单行截断最强；摘要单行灰字次之；标签 + 来源最弱。缩略图提供快速辨识，扫读效率最高。
- **交互元素摆放**：整行可点；hover 整行底色变化 + 左侧陶土橙条滑入；右侧图标悬停显形。适合键盘/屏幕阅读器线性浏览。
- **多屏适配**：移动端缩略图缩至 46px、操作收进行内；≥640 恢复完整三栏。文字统一 ellipsis 单行防抖动。

## 4. 时间轴（Timeline）

- **图标排版与布局**：左侧竖线 + 节点圆点（陶土橙）；月份分组标题 sticky 吸顶；每卡左接节点。
- **信息层级与视觉重点**：时间（月/日）为结构锚；标题次之；摘要/标签最弱。节点实心=置顶、描边=普通，一眼辨重点。
- **交互元素摆放**：点卡片进详情；点分组标题折叠该月（max-height 过渡）。适合「按时间回顾」如月度复盘。
- **多屏适配**：移动端竖线贴边、卡片占满；保持单列，避免双列割裂时间感。

## 5. 分组折叠（Grouped Accordion）

- **图标排版与布局**：组头左侧类别图标（32px 圆底按分类取色）+ 名称 + 计数；右侧 chevron 表折叠态，整条可点。
- **信息层级与视觉重点**：组 > 组内卡片；组头渐变底强调分区；组内卡片弱化边框、共享对齐。
- **交互元素摆放**：点组头展开/折叠（chevron 旋转）；组内卡片可拖拽重排（示意）。适合主题多、需归类的知识库。
- **多屏适配**：组内卡片移动端单列 → ≥768 两列 → ≥1024 三列；组头始终通栏。

## 6. 看板（Kanban）

- **图标排版与布局**：列头左侧状态圆点（按状态取色）+ 标题 + 计数；卡片左上拖拽手柄（⠿）。状态即主要视觉编码。
- **信息层级与视觉重点**：列（未整理/进行中/已归档）一级分区；卡内标题 > 摘要 > 标签；不同状态列用极淡色区分。
- **交互元素摆放**：横向滚动；卡片拖拽手柄换位（示意）；列尾「+ 新增」引导续写。适合写作流 / 工作流。
- **多屏适配**：移动端列宽 82vw 横滑 → ≥768 列宽 262px → ≥1024 多列并排；列头 sticky。

## 选型建议

| 场景 | 推荐 |
|---|---|
| 泛览、视觉吸引力 | 卡片网格 / 瀑布流 |
| 快速扫读、信息密度 | 列表 |
| 回顾、时间脉络 | 时间轴 |
| 主题多、需归类 | 分组折叠 |
| 写作流 / 工作流 | 看板 |

> 六种方案共用同一组件（卡片/缩略图/标签/图标按钮）、同一 Token、同一响应式断点，因此可任意组合落地（如「列表为主 + 看板为整理视图」），不会出现视觉撕裂。
