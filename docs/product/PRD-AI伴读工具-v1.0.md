# AI 伴读工具 · PRD v1.0（定稿）

> 状态：v1.0 定稿 · 需求探索阶段结束，进入计划评审
> 维护：Senior Developer（吴八哥）

## 1. 产品定义（锚定原始需求，已修正两轮误读）
导入 **PDF / EPUB** → 阅读 → **划线 / 标注** → 读不懂时**直接在旁边问 AI**（解释这句话 / 给延伸建议）。
把"读 + 划线 + 当场问 AI"做成一体、顺滑的伴读体验（手机为主，桌面侧栏）。

> 关键修正：用户**当前的个人知识管理流程**（微信读书划线 → 导出 Obsidian 整理 → 写作用 Claude 匹配）是*现状 workaround*，**不是产品定义**。产品要替代这套笨流程。

## 2. 目标用户 / 内容域
- v1 内容域：**长类文章 + 职场学习**（非技术为主；小说/成长类为个人延伸）
- 未来商用主目标：**职场学习人群**（付费意愿较强）
- 现用工具（背景）：微信读书 + Obsidian + Claude

## 3. 交互设计
- **桌面端**：侧边常驻对话栏（选中文字 → 侧栏 AI 基于上下文解答/建议；可就同一段继续对话）
- **手机端**：待定（设计阶段定；默认提议：底部上滑抽屉式 AI 面板 + 长按选中弹气泡菜单：划线 / 问 AI / 复制）
- 主战场：手机（微信读书习惯），桌面侧栏已明确

## 4. MVP 功能清单（v1）
- ✅ 书架（多本管理）
- ✅ 导入 PDF / EPUB
- ✅ 阅读 + 划线 / 标注
- ✅ 旁边问 AI（解释 + 给建议；桌面侧栏 / 手机抽屉）
- ✅ 导出：**Markdown 文件 + 直推 Obsidian**（走 Local REST API；缺配置时降级为下载 .md）

## 5. 非功能
- 手机优先体验（桌面侧栏明确）
- **架构按可商用 / 多用户预留**（账号、数据隔离；所有表带 user_id）
- AI key 不落前端（薄后端代理）
- premium 标准：light/dark/system 主题、玻璃拟态、响应式、60fps、无障碍 WCAG 2.1 AA

## 6. 暂不做（v1）
- 社区 / 分享 / 多人协作
- 书籍级 RAG / 向量检索（商用二期）
- Web 剪藏（v1 仅 PDF/EPUB 导入）
- 多端实时同步（架构留接口，v1 单设备即可）

## 7. 关键技术风险
- 🔴 PDF 精准划线持久化（PDF.js 文本层能选中，但跨缩放/重渲染映射回原位需 page+char offset，最难）
- 🟡 Livewire 与 epub.js 共存（阅读器作静态 Alpine/JS 组件，Livewire 只管数据接口）
- 🟡 Obsidian 推送依赖本地插件 + key（做成可选，降级下载）
- 🟡 LLM 上下文长度（只注入选中文本+章节标题+近几轮，不塞整本）
