# AI 伴读工具 · PRD v0.1（活文档，随对话迭代）

> 状态：v0.1 · 需求探索阶段（找未知工作法）
> 维护：Senior Developer（吴八哥）｜对话共建，未开工

## 1. 用户画像（已确认）
- 读者类型：**小说 / 个人成长 / 学习类（非技术文）**
- 使用场景：**手机为主 + 电脑**；当前**自用**
- 痛点排序（用户原话）：
  1. 读不懂时想**结合上下文通俗解读**，但手机上复制粘贴去问 AI 太麻烦
  2. 遇到**击中心境的文字**想留作日后写作，但过后就忘
- 一句话目标：**让"读到的知识真的被用起来" + "读不懂时随手问 AI"**

## 2. 当前做法（关键线索）
- 先对内容做「赞成 / 标记」→ 进入**个人知识库** → 写日记 / 文章时**按心境搜索匹配** → 复习复用
- 含义：用户已有"**捕获 → 检索 → 复用**"心智模型；产品应承接并强化这条链路

## 3. 产品定位假设（待验证）
- 不是"阅读器 + 答疑"，而是 **「阅读—收藏—复用」闭环的个人知识伴侣**
- 三块心智：**读书（带 AI 解读）｜收藏（可检索知识库）｜写作辅助（按心境/主题回流收藏）**

## 4. AI 角色假设（待验证）
- **角色 A · 讲解员**：结合上下文，用通俗语言解读选中句段（手机端一键，免去复制粘贴）
- **角色 B · 写作燃料**：收藏的句子在用户写日记 / 文章时，按今天心境 / 主题被**召回推荐**
- 暂不强求"导师 / 对手"角色（用户未提，留作未来）

## 5. MVP 最小假设（待确认）
- 读一本 EPUB → 选中文字 → 两个动作：
  ① **一键通俗解读**（自动带上下文）
  ② **一键收藏**（自动打心境 / 主题标签）
- 收藏汇总成**可搜索知识库**
- 是否需书架 / 多书管理：待确认

## 6. 非功能假设
- **手机是主战场 → MVP 必须移动端好用**（响应式优先，甚至移动优先）
- 自用为主 → 暂可单用户、本地 / 轻量存储

## 7. 待澄清（Round 2）
- 「赞成」具体指什么动作？当前个人知识库用什么工具（flomo / Obsidian / 语雀 / 备忘录…）？
- 未来形态：纯自用 / 给朋友 / 带社区的产品？决定现在是否预留多用户
- MVP 是否必须带书架 / 多书？还是先单本跑通闭环

## 8. 暂定「不做」（随 PRD 演进）
- 暂不强制多用户 / 社区 / 分享
- PDF 文本层划线（二期）
- 书籍级 RAG（二期，先看简单检索 + 心境标签）
