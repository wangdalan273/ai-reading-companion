# P12（章节总结 + 思维导图）+ Phase 0（OCR 基础件）· 实现验收与用法

> 本文是「11 项功能排期」里 **Phase 0 + Phase 1（P12）** 的落地验收与真机使用说明。
> 设计依据：`功能详细设计与排期-第一性原理-2026-07-08.md`。

## 一、为什么先做这两项（第一性原理回看）

- **Phase 0 · OCR**：阅读的前提是"有字可读"。扫描版 PDF 没文字层，后面所有 AI 功能（总结、概念图、人物图、跨书检索）都会失效。所以先把"本地 OCR 接入"作为地基。
- **P12 · 章节总结 + 思维导图**：线性阅读留下碎片，人脑以"结构"存知识。导图把碎片拼成地图，一眼看见"这本书讲了什么"，回头能定位章节。它是后面几乎所有图谱功能的"数据母矿"。

## 二、做了什么（文件清单）

**后端（纯 PHP，沙箱可验）**
- 迁移：`books` 加 `ocr_text` / `has_text_layer` / `mindmap_md` / `text_extracted_at`；新建 `chapters` 表（book_id, idx, title, source_text, summary, status, error, generated_at）。
- `app/Models/Book.php` + 新 `app/Models/Chapter.php`：关联与字段。
- `app/Services/LlmService.php`：新增 `complete()` 非流式三协议（openai/anthropic/gemini）+ `extractFull()` 取完整内容 + `mockComplete()` 离线演示。
- `app/Services/OcrService.php`：调本地 PP-OCRv6（默认 `http://127.0.0.1:8765/ocr`），服务不可达/超时**容错返回 null，不阻断上传**。
- `app/Services/BookTextService.php`：
  - EPUB：纯 PHP 解 zip → 读 OPF spine 顺序的 xhtml → 去标签抽章节文本（**无外部依赖**）。
  - PDF：探测 `pdftotext`（poppler）抽文字层；无则标记 `has_text_layer=false`。
  - `ocrPdf()`：用 Imagick 逐页转 PNG → 调 OcrService 补扫描版文本（需 Imagick + 本地 OCR 服务）。
- `app/Http/Controllers/AnalyzeController.php`：
  - `POST /api/book/{id}/analyze`：抽取章节 → 逐章 `LlmService::complete` 总结 → 聚合 markmap Markdown 存 `books.mindmap_md`。
  - `GET /book/{id}/mindmap`：返回状态 / 章节摘要 JSON。

- `resources/js/mindmap.js`：把脑图 Markdown 渲染成交互式 SVG（点节点缩放、拖拽）。
- `resources/views/book-mindmap.blade.php`：脑图页，含「🤖 生成 / 重新生成脑图」按钮。
- `routes/web.php`：加上述两路由。
- `dashboard.blade.php`：上传即触发文本抽取 + 卡片加「📊 脑图」入口。
- `read.blade.php`：阅读器工具栏加「📊 脑图」链接。

## 四、真机怎么用（3 步）

1. **填密钥**：「⚙️ AI 设置」里选厂商（OpenAI / Claude / Gemini 或自定义），填入 key。不填则走离线演示总结。
2. **导入书**：书架上传 PDF / EPUB，上传时会自动抽取章节文本（失败也不影响导入）。
3. **看脑图**：书架卡片点「📊 脑图」，或阅读器点「📊 脑图」→ 点「🤖 生成 / 重新生成脑图」，AI 逐章总结并画出全书结构图。

## 五、扫描版 PDF（OCR）怎么生效

你已经装好本地 PP-OCRv6（FastAPI @ `127.0.0.1:8765`）。要真正跑通扫描版补文字，还需满足：
- PHP 启用 **Imagick** 扩展（用于把 PDF 每页转 PNG）；
- OCR 服务在线。
满足后，上传扫描版 PDF → 无文字层 → 自动逐页 OCR → 补 `ocr_text` → 后续总结/导图照常工作。
（沙箱环境无 OCR 服务与 Imagick，该路径代码已写好并容错，但未实测；你真机直接可用。）

## 六、已知限制

- EPUB 章节切分按 spine 顺序（天然章节边界）；PDF 无书签时按 ~3000 字近似分段（标题"第 N 部分"）。
- PDF 文本层依赖 poppler 的 `pdftotext`；Windows 未装则降级为"待 OCR"。
- 真实逐章总结需要有效的模型 key；离线演示模式给的是模板化总结。
