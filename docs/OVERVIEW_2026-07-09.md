## 一、阶段2 伴读接口 bug 修复（此前自测漏网）

**Bug 1 — `addToKb` 报错 `Undefined array key "title"`**
- 根因：`$data['title'] ?: …` 在请求未带 `title` 时抛错——Laravel `validate()` 的 `nullable` 字段若请求未传，键本身不存在。
- 修复：`CompanionController::addToKb` 改为 `!empty($data['title']) ? $data['title'] : 兜底生成`。

**Bug 2 — `ask` 误报 422 `The message field is required`**
- 根因：`MergeJsonPayload` 中间件用 `empty($request->all())` 作为合并 JSON body 的前置条件，一旦请求带 query 参数或其它字段，合法 JSON body 被漏合。
- 修复：`app/Http/Middleware/MergeJsonPayload.php` 改为「`isJson()` 且 body 可解析即无条件 `add()` 到 json bag」，幂等且不再漏合。

**真 HTTP 端到端验证（全部 200）**
- `/api/companion/ask`：scope=all / 带 persona_id，SSE 流式逐字返回，以 `data: "[DONE]"` 收尾；伴读消息正确落库 `companion_messages`（persona_id 命中即记录，跨书 scope 下 book_id=null）。
- `/api/companion/add-to-kb`：带/不带 title 均 200，写入 `rag_chunks`（source_type=companion），并失效+重建知识库图谱缓存。
- 人格库 CRUD：store / update / destroy / list 全部 200。
