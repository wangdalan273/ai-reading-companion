## 修复一：选区浮条贴词显示（原「离得有些远」）

**现象对照**：之前选中一个词后，上方浮条（问 AI / 翻译 / 解释）离选中文字较远、位置飘。

**根因（第一性原理）**：阅读器正文在 epub.js 生成的 `<iframe>` 里，选区坐标是 iframe 内部坐标，必须叠加 iframe 在页面中的偏移（`getBoundingClientRect`）才能换算到页面视口坐标。之前直接用了 iframe 内部坐标，缺少这层偏移，导致浮条位置漂移。

**测试步骤**：
1. 在阅读页正文里，用鼠标选中一段文字（一个词或一句话均可）。
2. 观察浮条出现位置：应**紧贴选中文字正上方约 8px**，不遮挡选中内容。
3. 边界场景：
   - 选中内容**贴近阅读区顶部**：浮条自动翻到选中文字**下方**显示（不会被顶部裁掉）。
   - 选中内容**贴近视口左右边缘**：浮条自动向内侧 clamp，不会被屏幕裁掉一半。
4. 滚动页面后再选中：浮条位置应跟随选区实时正确（不残留旧偏移）。

**预期结果**：浮条稳定贴在选区上方（或顶部时下方），左右不越界，滚动后不错位。

## 修复二：左侧目录点选高亮（原「点了不知道当前哪个章节」）

**现象对照**：点目录跳转后能翻页，但目录里没有高亮，不知道自己现在在哪个章节下。

**测试步骤**：
1. 桌面端：阅读页左侧目录，点任意章节 → 正文跳到该章，且**被点的条目高亮**（浅蓝底 + 主色字 + 加粗）。
2. 翻页验证：用「点边翻页」或直接滚动跨章 → 目录高亮**自动跟随当前章节**变化（不必手动点）。
3. 手机端：底部上滑抽屉 → 目录，点条目跳转并高亮，选完抽屉自动收起。
4. 跨章连续性：从第 1 章一路翻到第 5 章，目录高亮应逐章正确迁移，不串章。

**预期结果**：目录始终明确标记「当前所在章节」，且翻页时自动同步，无需手动点。

**实现要点**：监听 epub.js `rendition.on('relocated')`，归一化 href 后匹配目录，派发 `companion:relocated` 事件；`read.blade.php` 用 `activeHref` 接收并给对应条目加 `:class` 高亮。

## 修复三：翻页热区不再吞文字选取（原「周围光显示小鼠标，选不了字」）

**现象对照**：阅读区左右边缘一片区域只显示小手指光标，鼠标移进去无法选中该区域的文字。

**根因（第一性原理）**：翻页热区原本 `pointer-events: auto` 常驻覆盖在边缘 8% 区域，截断了文字选取事件。改为默认 `pointer-events: none`（事件穿透到正文，可正常选字），**仅当鼠标真正贴边（≤8% 或 48px）时才激活**接管翻页，移开即恢复穿透。

**测试步骤**：
1. 把鼠标移到阅读区**最左/最右边缘约 8% 宽**的竖条区域：
   - 鼠标**不在边缘**（离边 >8%）：该区域**可正常选中文字**，光标是文本 I 型，不出现小手指。
2. 鼠标**贴边移动**（≤8% 宽度）：边缘区域激活，显示翻页提示（高亮/箭头），点击即翻页。
3. 移开边缘：立刻恢复成可正常选字的状态。
4. 手机端：点屏幕左/右 8% 边缘触发翻页；中间区域正常选词、滑屏。

**预期结果**：边缘区默认透传文字选取，只有真正贴边时才接管翻页，不再「一靠近边缘就选不了字」。

## N9：魔鬼代言人（挑刺模式）

**功能**：对话栏新增 🎯 开关。开启后，AI 不再附和，而是专门挑刺、挑战你的理解，帮你把想法打磨得更严谨。

**测试步骤**：
1. 打开右侧/底部 AI 对话栏。
2. 找到 🎯 **魔鬼代言人：关** 按钮，点一下 → 变成「开」，旁边提示「已开启：接下来我会专门挑刺」。
3. 在输入框写一段你自己的理解/观点（例如「我觉得这本书核心是说人要自律」），发送。
4. 观察回复：
   - **无 AI Key（默认离线）**：回复以「（魔鬼代言人·离线演示）既然要挑刺…」开头，内容主动质疑你的表述、给出相反视角。
   - **有 AI Key（应用内 AI 设置已配）**：回复由真实模型按「挑刺者」角色生成，温和但犀利地找漏洞。
5. 再点一下 🎯 关掉 → 恢复普通问答模式（不再挑刺）。

**预期结果**：开关切换即时生效；同一句话，开/关魔鬼模式给出的回复风格明显不同（挑刺 vs 正常解答）。

**实现要点**：`companion.js` 加 `devil` 开关与 `toggleDevil()`，发请求时带 `mode: 'devil'`；`CompanionController::ask` 校验并透传 `mode`；`LlmService::buildMessages` 在 `mode==='devil'` 时往 system prompt 注入「魔鬼代言人」角色指令；`mockStream` 离线分支返回挑刺演示文本。

## 出问题当场修

真机若发现以下任一，回来反馈，我按第一性原理定位：
- 浮条仍偏移 → 查 `_iframeBase()` 是否取到了 iframe 正确偏移。
- 目录不高亮/串章 → 查 `relocated` 的 href 归一化与 `activeHref` 绑定。
- 边缘仍吞选字 → 查 `.companion-tapzone` 的 `pointer-events` 与 `.active` 切换逻辑。
- 魔鬼模式无效 → 查 `mode` 字段是否从 `companion.js` 一路透传到 `LlmService`。
