## 一、测试范围与策略

### 1.1 测试环境
- **应用地址**：`http://127.0.0.1:8123`（本地 `artisan serve`，已在运行）
- **测试账号**：本机免注册账户 `local@companion.local`；访问 `/enter` 自动登录并跳转到书架
- **主题状态**：浅色 / 护眼 / 深色 三态（导航栏切换，存 `localStorage`）
- **AI 模式**：默认**离线演示**（未填密钥也能跑完整交互）；填密钥后走真实模型（受 CloudBase 免费额度限制，可能 429 降级）

### 1.2 测试方式
- **自动（我执行）**：用 `curl` + 会话 cookie 对 `8123` 做 HTTP 级校验（路由可达性、CSRF、JSON 返回、导出下载、索引/生成端点、越权 403/302 等），覆盖后端契约。
- **手测（你执行）**：纯浏览器交互（选词浮条、画布拖拽、语音朗读、视觉渲染）我无法在沙箱里点击，会给出**精确步骤 + 预期**，你在浏览器操作、我记录结果。
- 每完成一项，在「问题记录表」填：实际 vs 预期、复现步骤、严重级、状态。

### 1.3 严重级定义
- **P0 阻断**：功能完全不可用 / 报错 / 白屏 / 数据错误
- **P1 严重**：主路径可用但结果错误 / 丢失数据 / 越权
- **P2 一般**：次要路径异常 / 提示不友好 / 偶发
- **P3 轻微**：文案/样式/体验瑕疵
