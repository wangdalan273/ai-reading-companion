技术栈全景速查手册

从第一性原理看懂：前端有哪些、后端有哪些、数据库有哪些，以及它们怎么串成一条产品线。带团队做新项目时，照着选就不慌。

核心心法：先定目标 → 列能力 → 每项选"最小够用"的解。不要被潮流绑架，也不要为了"显得专业"上重武器。能一个人几天跑起来的方案，往往就是最好的方案。

一、一条主线：任何软件都长这样

不管你做网站、App 还是小程序，背后永远是一条链路：

客户端（你点的界面） → 网络接口（HTTP/REST 等） → 服务端（想业务逻辑） → 数据库（存数据）

此外还有两横一竖的"通用层"：横向支撑（登录鉴权、CDN 加速、缓存、消息队列）贯穿全程；工程化运维（构建、容器、CI/CD、云）负责把代码变成能上线的产品并让它稳稳跑着。

二、前端全家族（用户看得到、摸得着的那层）

类别干什么代表技术什么时候用
基础三件套网页的地基：结构 / 样式 / 交互HTMLCSSJavaScript永远都要，逃不掉
框架把界面拆成可复用"组件"，状态自动管理ReactVueAngularSvelte界面复杂、要交互、要可维护
元框架框架 + 服务端渲染 + 路由 + 打包一体Next.jsNuxtAstro要 SEO、要首屏快、要全栈一体
样式与组件库少写 CSS，直接拼出统一好看的界面TailwindFluxUIAnt Design想快出活、又不想丑
跨端一套代码跑手机 / 桌面 / 网页React NativeFlutterElectronTauriPWA不想为每个平台单独开发
特效与 3D动画、粒子、三维沉浸感Three.jsGSAP做"高级感"、产品展示、游戏化
我们这个项目（AI 伴读）用的就是：基础三件套 + Livewire/Volt（一种"写 HTML 就有交互"的特例框架，让前后端不分离）+ Tailwind + FluxUI + epub.js（专门渲染电子书）+ 自己写的 JS（手势/护眼/AI 面板）。

三、后端全家族（藏在服务器里想逻辑的那层）

语言代表框架风格与适合
PHPLaravelThinkPHP上手快、部署简单、生态老牌。做内容/工具类网站极顺手
Node.jsExpressNestJS前后端同用 JS，实时通信（聊天、推送）天然友好
PythonDjangoFlaskFastAPI写起来最省力，AI / 数据生态最强（做 AI 产品首选）
JavaSpring Boot企业级大厂标配，稳但重，适合超大规模团队
其他GoRustC#(.NET)Ruby高性能 / 强类型 / 老牌各有所长，按需取用

前后端怎么"对话"——API 协议

后端干完活，把结果按约定格式交给前端，这套约定叫 API：

REST最通用、最简单，网址即资源。90% 项目够用
GraphQL前端要什么字段自己说，适合复杂查询
gRPC服务间高速通信，内部微服务用
SSE / WebSocket服务器主动、实时往外推（如 AI 逐字输出、聊天）

四、数据库全家族（数据存哪）

类型代表适合
关系型 SQLMySQLPostgresSQLite要事务、要强关系（用户-订单-商品）。最稳最通用
文档 NoSQLMongoDB结构常变、海量半结构化数据（日志、素材）
键值Redis极快缓存、会话、排行榜。常配合 SQL 用
搜索Elasticsearch全文检索、模糊匹配（站内搜索、日志分析）
图Neo4j关系网络（社交、推荐、知识图谱）
选型口诀：要事务和关系 → SQL；要灵活和海量 → 文档；要快 → Redis；要搜 → ES；要关系网 → 图。自用小产品，一个 SQLite 就能起步，将来再升级。

五、工程化与运维（让产品能上线、能喘气）

构建工具Vite / Webpack：把零散代码压缩合并成浏览器能快速加载的文件
容器Docker：把环境和代码打包，换台机器也能一模一样地跑
CI/CD提交代码自动测试+部署，少人工、少出错
云AWS / 阿里云 / 腾讯云：服务器、数据库、存储按需租用
Web 服务器Nginx / Apache：真正对外接流量的"门面"（开发期可用自带轻量服务器）
包管理npm（前端）、Composer（PHP）、pip（Python）：装/管各种"零件库"

六、第一性原理选型框架

先问自己再决定
用户主要在哪用？手机浏览器 / App / 桌面网页→前端框架；真 App→RN/Flutter；都想→PWA
团队会什么、几个人维护？会 PHP→Laravel；会 JS→Node；要做 AI→Python
数据要事务和关系吗？要→SQL；不要且海量→文档；都要→SQL 为主+Redis 辅助
要实时推送吗（聊天/AI 流式）？要→SSE / WebSocket
现在几个人用，将来多少人？自用/小团队→SQLite+单机；要上线多人→MySQL+云
要快上线、还是要长期大工程？快→选最省事的一体化方案；长期→考虑可扩展性

七、常见套餐（直接抄）

① 轻量工具/内容站Laravel + Livewire + Tailwind + SQLite。一人几天上线
② AI 产品（含流式）Python(FastAPI) 或 Laravel + SSE + 任意前端 + SQLite/Postgres
③ 重交互 Web 应用React/Vue + Node(Express/Nest) + MySQL + Redis
④ 真原生 AppReact Native / Flutter + 后端 API + 云数据库
⑤ 搜索/数据平台任意后端 + Elasticsearch + 关系库做主存
⑥ 企业大系统Java(Spring) + Postgres + 微服务 + 容器编排

八、术语小词典

词人话
前端 / 后端前端=你看见的界面；后端=服务器里跑逻辑、存数据的部分
框架 vs 库库是被你调用的工具；框架是"它来调用你"，定好骨架你填肉
REST / API前后端约定的"对话格式"，网址即动作
SSE / WebSocket服务器能主动、实时把数据推给前端（不是前端一次次问）
SQL vs NoSQLSQL=表格、强关系、事务稳；NoSQL=灵活结构、易扩展
Docker / 容器把"代码+环境"打包成标准箱子，到哪都能原样跑
CI/CD代码一提交，自动测试并部署，减少人工失误
CDN把图片/脚本缓存到离用户最近的节点，加载更快
PWA网页伪装成 App，可加到手机主屏、离线可用
Vite / Webpack构建工具：把开发时的散文件，打包成上线用的高效文件
