## 二、关于"搜索 gitw 有没有可直接用的"

按你的要求查了 GitHub 上 AI 对话 → Obsidian 的成熟方案（AI Exporter / ChatGPT-to-Obsidian 等），共识是：
**干净 Markdown + Frontmatter 元数据 + 代码块/LaTeX 保留 + Obsidian callout（`> [!note]`）**。

本项目的 `ExportService` 本来就有 Frontmatter + 划线导出能力，**直接复用、补齐对话段落即可**，不必引第三方库（避免增加维护面与许可风险）。对话导出采用 `> [!question]`（我）/ `> [!note]`（AI 伴读）callout，魔鬼代言人轮次标「🎯魔鬼代言人」——导入 Obsidian 后渲染美观、可双向链接。

## 四、真机测试步骤（请你验）

1. 打开 http://127.0.0.1:8123 → 进入一本书的阅读页。
2. **场景 A（选区问 AI）**：选中一句 → 浮条点「💬 问 AI」→ 面板弹出、自动聚焦、AI **流式解读且不再出现 `[DONE]` 乱码**，自动滚底。
3. **场景 B（手动发）**：输入框打字发送 → 流式回复正常显示；若撞 CloudBase 限流会自动降级离线演示（面板不死）。
4. **场景 C（魔鬼代言人）**：点 🎯 开关 → 提问后 AI 转挑刺口吻；导出时该轮标「🎯魔鬼代言人」。
5. **历史回显**：关闭阅读页再重新打开同一本书 → 之前聊过的内容应**自动出现**在面板里。
6. **导出对话**：阅读面板头部点「📤 对话」（或书架页点「对话 MD」）→ 下载一个 `.md`；用 Obsidian 打开，应看到 Frontmatter + 每轮 `> [!question]`/`> [!note]` 的干净对话；魔鬼轮有标注。书架页「导出 MD」现在也**内含 AI 对话记录**段落。

## 五、已知限制（不是 bug）

- 你的 **CloudBase 免费额度极紧**，连续多次调用很快 429。此时对话会降级离线演示回复、图谱/脑图降级离线生成——App 绝不假死；**额度恢复后下一次自动回真实模型**。
- 历史回显只加载本书对话；跨书汇总暂无（如需可后续加「全部对话」视图）。
- 对话导出是**下载 .md**；若你配了 `COMPANION_OBSIDIAN_VAULT` 环境变量，「推 Obsidian」按钮会把含对话的整本笔记直接写进 vault 文件夹（现有能力，本次让笔记自动包含对话）。
