# N6 论证地图 · 真机测试文档（2026-07-09）

> 第一性原理：非虚构阅读的本质不是「记住」而是「判断」——作者凭什么这么断言？证据够不够？有没有考虑反面意见？N6 把隐含的论证结构显式画出来，培养批判性思维。

## 一、功能概览

「⚖ 论证地图」从全书抽取 **主张—证据—反驳** 的论证骨架，渲染成可交互的关系图，并给每条主张附一句 **批判性质询**（逼读者思考反例与前提假设，呼应 AMQuestioner 的「用生成问题练批判性思维」思路）。

- **主张节点**（琥珀色大圆）：作者断言的核心论点。
- **证据节点**（绿色小圆，实线相连）：支撑该主张的数据 / 案例 / 引用 / 推理。
- **反驳节点**（红色菱形，虚线相连）：对该主张的反方观点、局限或反驳。
- 点任意节点看详情；侧栏「全部主张」可逐条浏览；统计条显示主张 / 证据 / 反驳数量与体裁判定。

## 三、抽取逻辑

- **真实模型路径**：分章调 `LlmService::complete`，要求模型按 DeepA2 风格输出 `claims / evidence / counter` JSON，并给每条主张附 `challenge` 批判性质询；跨章合并去重，统一补齐质询。
- **离线兜底路径**：扫描分句，用断言标记词（「我认为 / 总之 / 关键在于 / 应该」等）识别主张，用证据标记词（「例如 / 数据显示 / 研究表明」等）识别证据，用反驳标记词（「然而 / 相反 / 有人认为」等）识别反驳，把相邻证据 / 反驳归到最近的主张下，形成可演示的论证骨架。`ensureChallenges()` 为所有主张统一补一句质询。

## 五、真机验收步骤

1. 浏览器打开 `http://127.0.0.1:8123/`（或 `/enter` 免注册直接进入书架）。
2. 打开一本 **非虚构类** 书（如《零起点学中医》），点阅读页顶部 / 侧栏的 **「⚖ 论证」**。
3. 点右上角 **「🤖 生成 / 重新生成」**，等待数十秒～数分钟（首次抽全书论证，分章多次调用模型）。
4. 预期看到：
   -  amber 大圆 = 主张，绿小圆（实线）= 证据，红菱形（虚线）= 反驳；
   - 点任一主张 → 右侧显示完整论述 + 🤔 批判性质询 + 支撑它的证据与反驳列表；
   - 侧栏「全部主张」可逐条点击聚焦；
   - 顶部统计条显示主张 / 证据 / 反驳数量与体裁判定。
5. 拖拽节点排版、滚轮缩放、拖背景平移。
6. 下次进同一本书直接显示已生成地图（落库持久化）。

## 六、离线 vs 真实

- **未填密钥 / 429 限流**：自动降级为离线论证骨架（断言 / 证据 / 反驳标记词启发式），质询为通用模板。
- **填有效密钥且额度充足**：真实模型抽取精准主张 / 证据 / 反驳，并为每条主张生成针对性批判性质询。

## 七、已知限制与后续增强

- 离线兜底的「主张」识别依赖标记词，长难句可能漏抽；真实模型路径质量更高。
- 设计里原拟「点主张让 AI 实时补刀反驳」的交互式增强，本轮以生成时预置 `challenge` 质询体现批判性，如需「实时补刀」可在下一轮接入 `companion.ask` 流式。
- Phase 2 已全部完成（N3 概念图谱 / N7 人物关系 / N6 论证地图）。下一步 Phase 3：P13 跨书 RAG + 自定义 prompt（N12 知识库图谱同属 Phase 3）。
