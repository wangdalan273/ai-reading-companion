## 1. 第一性原理：阅读认知价值链

人读书，本质是五个递进的动作。每一层都**依赖上一层先把数据准备好**，所以功能必须**自下而上**推进：

| 层 | 第一性原理（人为什么需要它） | 我们现状 | 这一层缺什么 |
|---|---|---|---|
| **① 接入层** | 阅读的前提是"有字可读、可读的部分能被定位"。扫描版没文字层、图表里的信息纯文本抓不到，AI 就瞎了。 | ✅ EPUB/PDF 导入、划线持久化 | OCR、图表/公式多模态理解 |
| **② 理解层** | 遇到"读不懂/有感触/想深挖"的地方，希望有个**懂这本书**的伙伴在旁边即时解读。 | ✅ 旁边问 AI（P11 后已全厂商开放） | 结构化理解（总结/导图/概念图/批判）、外部资料增强、语音输入 |
| **③ 记忆层** | 人的遗忘曲线决定：不主动提取/复习，划线等于白划。把感触沉淀成可复用卡片才留得住。 | △ 手动闪卡、词汇本（规划中）、阅读时长打卡 | 自动生成闪卡、导出 Anki |
| **④ 复用层** | 真正的学习发生在"跨书连接"——读过的书之间互相印证，知识才长成网。 | △ 导出 Markdown/Obsidian、手动闪卡 | 跨书 RAG、个人知识库图谱、双向链接 |
| **⑤ 延展层** | 阅读不止用眼睛：耳朵（听）、对照（外语）、生成（输出）都是"读"的延伸形态。 | △ TTS 朗读、金句卡 | 音频概览、双语对照、个性化荐书、语音输入 |

> 关键推论：**越靠后的层价值越高、但越依赖前面的结构化数据**。所以"先做数据基建，再做炫酷产出"是铁律。

## 3. 逐项详解

### ① 接入层（让"内容"先变得可被 AI 读懂）

#### N1 · OCR 扫描版 PDF 文字层
- **第一性原理**：阅读的前提是"有字"。很多书（尤其老书、教材、扫描档案）是图片式 PDF，没有文字层，划线、搜索、AI 理解全失效。
- **做什么**：上传 PDF 时检测是否缺文字层；缺则自动跑 OCR（本地 PaddleOCR / 云端）补一层可选中文字，再进现有流水线。
- **GitHub 证据**：`OCRmyPDF`（OCR topic 顶流，80+ 语言）、`askitlouder/Document-Scanner-And-Summarization`（扫描件→文字→总结）。
- **依赖**：OCR 引擎（本地部署较重，或接 OCR API）。**工作量：中**。**风险：中**（OCR 质量参差、中文古籍易错；本地引擎吃资源）。

#### N2 · 图表/表格/公式多模态理解
- **第一性原理**：书里 30%+ 的关键信息藏在图、表、公式里，纯文本 RAG 会整块漏掉——而读者最卡壳的往往正是这些。
- **做什么**：对 PDF 里的图片/表格/公式做多模态嵌入；问 AI 时把"当前图"连同问题一起发给视觉模型，给出解读。
- **GitHub 证据**：`miketybear/multimodal-rag`、`jiangnanboy/pdf_multimodal_rag`、`RAG-Anything`（港大，文本+图+表+公式端到端）。
- **依赖**：多模态模型（Vision，如 Gemini/Claude/GPT-4o）。**工作量：大**。**风险：高**（解析复杂、成本高、EPUB 图表提取更难）。

### ② 理解层（让 AI 真正"帮你想通"）

#### P12 · 章节级 AI 总结 + 全书思维导图
- **第一性原理**：线性阅读留下的是碎片；人脑以"结构"存储知识，导图/大纲把碎片拼成地图，回头能一眼看见"这本书讲了什么"。
- **做什么**：按 EPUB 章节切分，逐章生成总结，再聚合出全书思维导图（可点节点跳回原文）。
- **GitHub 证据**：`ai-reader-pro`、`ebook-to-mindmap`、`chapterAI`（分章节总结是共性需求）。
- **依赖**：P11。**工作量：中**。**风险：低**。

#### N3 · 知识图谱 / 概念图谱（自动连线）
- **第一性原理**：比导图更进一步——概念之间不是树状而是网状。"A 和 B 什么关系""这个概念在哪一章出现"用图比用大纲更贴近人脑。
- **做什么**：从全书抽取"主语-关系-宾语"三元组，渲染成交互式概念图，点节点看定义与出处。
- **GitHub 证据**：`robert-mcdermott/ai-knowledge-graph`、`openimpactai/KnowledgeGrapher`、`KGGen`（斯坦福）。
- **依赖**：P12 的章节总结（先有可抽取的文本单元）。**工作量：中–大**。**风险：中**（图谱质量依赖模型抽取能力，需做去噪）。

#### N4 · 难度可调总结（ELI5 / 中学生 / 专家级）
- **第一性原理**：同一段内容，不同读者的认知负荷不同。能一键切换"讲给小孩听 / 讲给同行听"，是个性化理解的核心。
- **做什么**：总结/解读时提供"通俗/标准/深入"三档，复用同一 prompt 模板切换语气深度。
- **GitHub 证据**：社区通用 pattern（无单一代表 repo，但"explain like I'm 5"是 LLM 经典用法）。
- **依赖**：无。**工作量：小**。**风险：低**（纯 prompt 层）。

#### N5 · 术语/概念悬停解释（划词即查）
- **第一性原理**：阅读中断成本极高——遇到生词/陌生概念卡住，是读者放弃的头号原因。零步成本地"就地解释"能把中断降到最低。
- **做什么**：选中一个词/术语，浮条加「📖 释义」，派发解释请求（复用现有划词问 AI，但预置"用一句话通俗解释这个词"）。
- **GitHub 证据**：沉浸式翻译/划词类工具的核心交互；Koodo Reader 的"划词翻译/解释"。
- **依赖**：P11。**工作量：小**。**风险：低**（复用选区基础设施）。

#### N6 · 论据/论证地图（非虚构批判阅读）
- **第一性原理**：非虚构阅读的本质不是"记住"，而是"判断"——作者凭什么这么断言？证据够不够？把隐含的论证结构显式画出来，培养批判性思维。
- **做什么**：对非虚构书/章节抽取"主张-证据-反驳"，渲染成论证地图；可点"这个说法的反面证据是什么"。
- **GitHub 证据**：`argumentation` topic（DeepA2 等）、`AMQuestioner`（CSCW'25，用生成问题练批判性思维）。
- **依赖**：P12。**工作量：中**。**风险：中**（论证抽取对模型要求高，长文易漂移）。

#### N7 · 小说人物关系图 + 时间线
- **第一性原理**：小说读者最大痛点是"人物太多记不住、时间线乱"。把人物关系与时间线显式画出来，沉浸感与回顾效率倍增。
- **做什么**：识别书中人物，抽取关系（亲属/敌对/合作）与关键事件时间线，渲染成可交互关系图 + 时间轴；点人物看其出场章节。
- **GitHub 证据**：`Drwei3155/story-graph`、`yumeng001yu/ai-novel-character-graph`、`Austen`（Mermaid 角色关系图）。
- **依赖**：体裁识别（小说 vs 非虚构，可由 AI 判断）。**工作量：中**。**风险：中**（人物共指/别名消歧是难点）。

#### N8 · 外部资料增强 / Grounding（AI 帮你查背景）
- **第一性原理**：书里提到的概念/事件/人物，读者常缺乏背景。让 AI 在解读时**联网查权威资料**补全上下文，等于给阅读配了个研究员。
- **做什么**：解读/总结时可开关"联网增强"，AI 调用搜索 API 取外部资料并标注来源，回答里带引用链接。
- **GitHub 证据**：`careed23/Grounded-Research-Assistant`（Gemini + Google Search Grounding）、`nav9v/gemini-search-mcp`。
- **依赖**：搜索 API（如 SerpAPI/Tavily/厂商自带 grounding）+ 联网。**工作量：中**。**风险：中**（成本、延迟、隐私——需默认关闭、用户明示开启）。

#### N9 · 辩论 / 魔鬼代言人（AI 反驳你的理解）
- **第一性原理**：深度理解需要"对抗性检验"——你以为懂了，让 AI 站在反面挑刺，漏洞立刻显形。这是主动思考的催化剂。
- **做什么**：对话栏加「🎭 反驳我」按钮，AI 针对你的观点/笔记生成反方论证，引导你修正。
- **GitHub 证据**：批判思维/辩论教练类 prompt 范式；与 N6 论证地图互补。
- **依赖**：无。**工作量：小**。**风险：低**（纯 prompt 层，可复用对话基础设施）。

### ③ 记忆层（把感触留下来）

#### P10（已有）手动闪卡 / P14（规划）词汇本 / 阅读时长打卡 —— 见前文。

#### N10 · 自动闪卡生成（划线 → 问答卡）
- **第一性原理**：手动做卡有摩擦；而"从划线自动出问答"把摩擦降到零，配合间隔重复（P10 已有 Leitner）， retention 大幅提升。
- **做什么**：选中一段，浮条加「🃏 AI 出题」，模型据原文生成一张"问题/答案"卡（Cloze 或 Q&A），直接进闪卡库。
- **GitHub 证据**：`VastHornet/anki-flashcards-builder`、`athravseruwam07/ai-flashcard-generator`（文本→可编辑问答卡→导出 Anki）。
- **依赖**：P10 闪卡表（已存在）。**工作量：小–中**。**风险：低**。

#### N11 · 导出到 Anki（.apkg）
- **第一性原理**：很多用户已有 Anki 复习习惯/牌组生态，导出能无缝接入，而不是另起炉灶。
- **做什么**：闪卡库支持一键导出为 Anki 可导入的 `.apkg` / `.csv`。
- **GitHub 证据**：`anki-flashcards-builder` 明确支持导出 CSV/Anki；AnkiDecks 支持 `.apkg`。
- **依赖**：P10/N10。**工作量：小**。**风险：低**（有成熟库 genanki）。

### ④ 复用层（让知识长成网）

#### P13 · 跨书 RAG 问答 + 自定义 Prompt —— 见前文（已排期）。

#### N12 · 个人知识库图谱 / Zettelkasten 原子笔记
- **第一性原理**：卢曼"卡片盒"方法——把笔记拆成原子单元并互相链接，连接本身会产生新想法。这是"伴读"走向"第二大脑"的关键一跃。
- **做什么**：每张划线/笔记是一张原子卡；AI 自动建议"这张和哪张相关"并连边；提供全局知识图谱视图。
- **GitHub 证据**：`joshylchen/zettelkasten`（CEQRC 工作流）、`Dzeddy/zettelkasten`（自动 chunk + 语义 embedding + 向量检索 + 互联图谱）。
- **依赖**：P13 的向量检索基建。**工作量：大**。**风险：中–高**（需 embedding/向量库，图谱质量调优）。

#### N13 · 双向链接笔记（Obsidian 式 `[[概念]]`）
- **第一性原理**：写作/笔记时手动 `@关联` 一个概念，等于在知识网里亲手牵一条线，比 AI 自动连更可控、更"属于自己"。
- **做什么**：笔记编辑器支持 `[[概念]]` 语法，自动建反向链接，点链接跳转；导出时转成 Obsidian 双链。
- **GitHub 证据**：Obsidian 核心交互；`Dzeddy/zettelkasten` 的互联笔记。
- **依赖**：P3 导出（已存在）。**工作量：小–中**。**风险：低**。

### ⑤ 延展层（阅读不止用眼睛）

#### P16 音频概览 / P14 双语对照 / P10 TTS —— 见前文（已排期/已有）。

#### N14 · 语音输入问 AI（说而不是打字）
- **第一性原理**：手机上打字问 AI 效率低、打断阅读心流；语音是更自然的接口，尤其通勤/躺读场景。
- **做什么**：对话栏加麦克风按钮，本地/云端 STT 把语音转文字再发问；也可语音控制翻页。
- **GitHub 证据**：`Angana007/AI-Reading-Assistant`（voice control + ask）、`whisper.cpp`（本地离线识别）。
- **依赖**：STT（浏览器原生 Web Speech API 即可起步，或 Whisper）。**工作量：中**。**风险：中**（中文识别准确率、移动端权限）。

#### N15 · 个性化荐书（基于你的书架 + 笔记画像）
- **第一性原理**：读什么，比怎么读更决定一个人的成长。基于"你读过/划线/问过什么"画像荐书，比通用榜单精准得多。
- **做什么**：分析书架标签 + 划线主题 + 问答历史，让 LLM 生成"下一本读什么"及理由；可接书目元数据。
- **GitHub 证据**：`ilzrzls/Book-Recommendation`（在线阅读&荐书&AI写作）、`akanksharai01/Book-recommendation-system`（基于阅读历史个性化推荐）。
- **依赖**：书目元数据来源（可选）。**工作量：中**。**风险：低–中**（纯 LLM 生成理由即可起步，无需复杂推荐算法）。
