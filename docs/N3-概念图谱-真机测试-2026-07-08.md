## 测试步骤

### 入口
- 阅读器页：桌面右侧 AI 栏顶部点 **🕸 图谱**；手机工具栏点 **🕸 图谱**（与 📊 脑图 并列）。
- 或脑图页（📊 本书脑图）顶部点 **🕸 概念图谱** 互跳。

### 1. 查看图谱（已有图谱时）
1. 进入 `/book/{id}/graph`，应看到一张**力导向交互图**：
   - 节点大小 ∝ 该概念在全书出现的频次；颜色按概念哈希区分。
   - 边表示「共现 / 关系」。
2. 默认自动收敛布局（约 1–2 秒稳定）。

### 2. 交互
- **拖节点**：鼠标按住某节点拖动，可手动排版；松手后周围节点会重新找平衡。
- **滚轮缩放**：以光标为中心放大/缩小。
- **拖背景平移**：在空白处拖动可平移整张图。
- **点节点**：右侧面板显示「概念名 / 出现频次 / 释义 / 出处章节（第几章）」。
  - 离线（未填密钥）模式释义显示占位提示；填密钥后由真实模型生成一句话释义。
- **悬停高亮**：鼠标移到节点上，其相邻节点与边高亮，其余淡出。

### 3. 生成 / 重新生成
1. 点右上角 **🤖 生成 / 重新生成**。
2. **离线（默认，无密钥）**：后端用「高频词 + 同句共现」启发式秒级生成（无需联网/密钥），即可见图。
3. **已填 AI 密钥**：后端逐文本块让真实模型抽取「主语-关系-宾语」三元组 + 定义，再合并归一化。
   - 首次可能数十秒～数分钟（取决于章节数与模型速度），进度条文案会提示「正在抽取…」。
   - 生成完自动渲染。

### 4. 边界情况
- 本书**从未生成过图谱**：页面提示「点右上角生成」，点生成即可。
- 节点过多时自动截断为 **前 90 个高频概念 / 200 条关系**，保证画布可读、流畅。
- 书无正文（扫描版且未起 OCR）：点生成会返回友好提示，不会崩。

## 出问题当场修

- 进页面空白/报错：先看浏览器控制台与终端 serve 日志；多半是 `public/build` 缺失 → 重新 `npm run build`（受管 node 路径）。
- 图谱一直转/不出：查 `books.concept_graph_status` 是否为 `failed`（看 `concept_graph_error`）；离线模式应秒出，若卡住多为模型调用超时（填密钥场景，可加 `set_time_limit` 已加）。
- 节点点不开详情：查 `showDetail` 是否拿到 `nodes[i]`；多为 `graph.nodes` 为空导致。
- 关系太密看不清：调 `finalize()` 的截断阈值（90/200）或力导向 `rep`/`spring` 参数。
