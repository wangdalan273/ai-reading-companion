## 一、问题现象（用户截图描述）

1. **之前 AI 回复的都显示了**（旧对话从历史加载正常渲染）
2. **但新提问后 AI 回复又不显示了**（新流式响应在 UI 中不可见）
3. **用户消息只显示 `;`**（内容几乎看不到）
4. **出现两个几乎相同的 AI 气泡**
5. **不能所有问答都集中在一个对话框里**（对话体验不统一）

## 二、根因分析（第一性原理）

### 核心缺陷：Alpine.js Vue 3 响应式陷阱

**旧代码** (`app.blade.php` `_stream()` 方法):
```javascript
// 第 1 步：创建普通 JS 对象
const assistant = { role: 'assistant', content: '' };

// 第 2 步：推入 Alpine 响应式数组 → Alpine 内部创建 Proxy 包装
this.messages.push(assistant);

// 第 3 步：后续通过原始引用修改内容 ← ★ BUG 所在
assistant.content += token;  // 改的是裸对象！模板读的是 Proxy（永远空字符串）
```

**为什么旧消息能显示？**
- `loadHistory()` 使用 `this.messages = data.messages.map(...)` — 整个数组替换，Alpine 完全追踪 ✅
- `_stream()` 使用「推入 + 裸引用修改」— 响应式链断裂 ❌

**Alpine 底层机制**: Alpine.js v3（Livewire 4 内置）使用 Vue 3 的 `@alpinejs/reactivity`。当普通对象被 push 进响应式数组时：
1. 数组的 Proxy 拦截 `push` 操作
2. 对被 push 的对象调用 `reactive()` 创建新 Proxy
3. 但**局部变量仍指向原对象**！后续修改原对象不会触发 Proxy 的 set 拦截器

### 仿真证据

```
❌ 旧方式 bare.content += 'x'   → 无 Proxy SET 输出 → 模板不更新
✅ 新方式 arr[idx].content += 'y' → ✓ Proxy SET [content] → 模板更新！
```

## 三、修复方案

### 3.1 核心：走 Proxy 链写入（app.blade.php）

```javascript
async _stream(message, context) {
    this.messages.push({ role: 'user', content: message, context: context || '' });
    this.messages.push({ role: 'assistant', content: '' });
    const aiIdx = this.messages.length - 1;     // ★ 索引捕获
    // ...
    // 所有写入改用 this.messages[aiIdx].content += token;
    // 错误信息也走 Proxy: this.messages[aiIdx].content = '错误...';
}
```

### 3.2 SSE 解析加固

- `[DONE]` 结束帧精确匹配（不做 trim，避免误杀含空格的合法 token）
- 兼容 JSON 字符串形式 `"token"` 和裸文本

## 六、技术要点备忘

- **Alpine/Vue 3 响应式铁律**：push 进响应式数组的对象，后续必须通过**索引访问**修改属性，不能用 push 时保存的局部变量引用
- 此陷阱在 React 中不存在（React 用 setState），是 Vue 3/Alpine 特有的 Proxy 机制坑
