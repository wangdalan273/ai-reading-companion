# 上云部署经验手册（AI 伴读 · read.sxmnq.art）

> 本手册沉淀自 2026-07-10 真实部署过程。面向「非专业运维、但要把 Laravel 应用稳稳放到已有域名的服务器上」的团队。
> 原则：**不碰老项目、子域名隔离、HTTPS 必须、可复制粘贴、出错有对策**。

---

## 一、整体架构（先想清楚再动手）

```
已有服务器（腾讯云轻量，Ubuntu 22.04，公网 IP 129.226.83.195）
├── 老项目：sxmnq.art（墨语手写模拟器，Node+pm2，前端在 /var/www/html）
└── 新项目：read.sxmnq.art（AI 伴读，Laravel+PHP-FPM，目录 /var/www/companion）
        两个项目：独立目录 / 独立进程 / 独立数据库 / 独立子域名 → 互不干扰
```

**为什么用子域名而不是子目录？**
Laravel 的路由、资源、`.htaccess`、前端构建产物都假定在域名根路径。塞进 `sxmnq.art/companion` 这类子目录会踩一堆路径坑。子域名是零成本、最干净的隔离方式。

**为什么不开 Docker？**
个人量级 + 单服务器，裸 Nginx + PHP-FPM 足够，少一层复杂度 = 少一层出问题的地方。

## 三、Step 1 — 加 DNS A 记录（最先做）

1. 腾讯云控制台 → 轻量应用服务器 → **域名**（或「轻量域名」）。
2. 找到 `sxmnq.art` → 添加记录：
   - 主机记录：`read`（系统自动拼成 `read.sxmnq.art`）
   - 记录类型：`A`
   - 记录值：`129.226.83.195`
   - TTL：600（默认）
3. 等 10 分钟，本地验证：
   ```bash
   nslookup read.sxmnq.art
   # 返回 Address: 129.226.83.195 即生效
   ```
> ⚠️ 证书申请依赖域名先解析通，所以 DNS 必须在 certbot 之前。

## 四、Step 2 — 服务器装 PHP 8.4（老项目是 Node，服务器原本没 PHP）

```bash
# 加 ondrej PPA（Ubuntu 装新版 PHP 的标准源）
sudo add-apt-repository -y ppa:ondrej/php
sudo apt update

# 装 PHP 8.4 + 必要扩展 + FPM
sudo apt install -y php8.4 php8.4-fpm php8.4-cli \
  php8.4-sqlite3 php8.4-curl php8.4-mbstring php8.4-xml \
  php8.4-zip php8.4-gd php8.4-bcmath php8.4-opcache php8.4-readline

# 装 Composer（用 php8.4 跑安装器）
php8.4 -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php8.4 composer-setup.php --install-dir=/usr/local/bin --filename=composer
php8.4 -r "unlink('composer-setup.php');"

# 确认
php8.4 -v
composer --version
sudo systemctl status php8.4-fpm   # 应 active (running)
```

> 装好后记下 socket 路径：`/run/php/php8.4-fpm.sock`（Nginx 的 `fastcgi_pass` 要用）。

## 五、Step 3 — Nginx 站点配置（与老项目平行，互不 include）

把下面内容写进 `/etc/nginx/sites-available/companion`，再软链启用：

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name read.sxmnq.art;

    root /var/www/companion/public;
    index index.php;

    client_max_body_size 512M;

    access_log /var/log/nginx/companion.access.log;
    error_log  /var/log/nginx/companion.error.log;

    location ~ /\.(?!well-known).* { deny all; return 404; }

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/run/php/php8.4-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        fastcgi_hide_header X-Powered-By;
    }

    # 静态资源：真实文件直接返回并缓存；虚拟 JS 路径(如 /livewire/livewire.min.js、
    # /flux/flux.min.js 由 Laravel 路由动态吐出)必须回退给 PHP，否则 404 导致 Livewire 失效。
    location ~* \.(?:css|js|png|jpg|jpeg|gif|ico|svg|woff2?|ttf|eot|webp|avif)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        try_files $uri /index.php?$query_string;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/companion /etc/nginx/sites-enabled/companion
sudo nginx -t          # 必须 test is successful
sudo systemctl reload nginx
```

> 先放一个占位 `public/index.php`（`<?php echo "companion ok";`）便于后续验证 HTTP 通了再上证书。

## 六、Step 4 — HTTPS 证书（certbot，绕过一个编码坑）

**坑**：服务器默认 locale 与 certbot 交互输入不兼容，填邮箱那一步会 `UnicodeDecodeError` 崩溃。
**解法**：跳过交互，参数全带上：

```bash
sudo apt install -y certbot python3-certbot-nginx

sudo LC_ALL=C.UTF-8 certbot --nginx -d read.sxmnq.art \
  --email 你的邮箱@qq.com \
  --agree-tos \
  --no-eff-email \
  --redirect \
  --non-interactive
```

成功后会自动：
- 在 `/etc/nginx/sites-enabled/companion` 加 443 server block
- 把 80 自动跳转 443
- 后台定时续期（证书 90 天，到期自动续）

验证：浏览器开 `https://read.sxmnq.art` → 小锁 🔒 + `companion ok`。

## 七、Step 5 — 上传代码（免密 SSH + tar 直传，最省事）

### 7.1 让运维机免密码连服务器
在**服务器**终端执行一条（把本机公钥粘进去）：
```bash
echo 'ssh-ed25519 AAAA...你的本机公钥...' >> ~/.ssh/authorized_keys
```
本机公钥在 `~/.ssh/id_ed25519.pub`（没有就 `ssh-keygen -t ed25519` 生成）。
> 好处：之后所有上传、装依赖、迁移都能在本地一键跑完，不用反复登服务器敲命令。

### 7.2 本地打包直传（排除大目录，体积小速度快）
```bash
# 本机 Git Bash / 终端执行
tar czf - --exclude=vendor --exclude=node_modules --exclude=.git \
  --exclude=.env --exclude='*.log' \
  -C /d/03_DevData/Projects/ai-reading-companion . \
  | ssh ubuntu@129.226.83.195 'tar xzf - -C /var/www/companion'
```
> 没 rsync？没关系，tar 管道一行搞定，且排除了 `vendor/node_modules/.git`（这些服务器会重建/不需要）。
> ⚠️ 注意：`.env` 被排除（服务器另建）；但 `database/database.sqlite` **没排除**会被一起传上去——如果本地库有调试数据，线上库就会带这些数据（功能无碍，要干净就删了重 migrate）。

## 八、Step 6 — 服务器初始化（顺序很重要！）

```bash
ssh ubuntu@129.226.83.195 'cd /var/www/companion && \
  cp -n .env.example .env && \
  composer install --no-dev --optimize-autoloader && \
  php8.4 artisan key:generate && \
  sed -i "s#^APP_URL=.*#APP_URL=https://read.sxmnq.art#; \
          s#^DB_DATABASE=.*#DB_DATABASE=/var/www/companion/database/database.sqlite#; \
          s#^APP_ENV=.*#APP_ENV=production#; \
          s#^APP_DEBUG=.*#APP_DEBUG=false#" .env && \
  touch database/database.sqlite && \
  php8.4 artisan migrate --force && \
  sudo chown -R www-data:www-data storage bootstrap/cache database && \
  sudo chmod -R 775 storage bootstrap/cache database && \
  php8.4 artisan config:clear && \
  php8.4 artisan route:clear && \
  php8.4 artisan storage:link'
```

> **致命顺序坑**：`key:generate` 必须在 `composer install` **之后**！否则 `vendor/autoload.php` 还没生成，artisan 直接 `Failed opening required '.../vendor/autoload.php'` 崩溃。本手册顺序已校正。

## 十、常见坑速查表

| 现象 | 原因 | 对策 |
|---|---|---|
| `rsync: command not found` | Windows Git Bash 默认无 rsync | 改用 tar 管道直传（见 7.2） |
| Git Bash 粘不上命令 | 用了 Ctrl+V（Git Bash 不认） | 右键粘贴 / `Shift+Insert` |
| certbot `UnicodeDecodeError` | 服务器 locale 与交互输入冲突 | 加 `LC_ALL=C.UTF-8` + `--non-interactive`（见六） |
| `Failed opening required vendor/autoload.php` | `key:generate` 跑在 `composer install` 前 | 调顺序：先 composer 后 key（见八） |
| 页面白屏 / 布局塌成单列 | 改了 blade 没重编前端 | 本地 `npm run build` 后重新上传 `public/build` |
| 上传附件 413 | 上传大小超限 | `client_max_body_size 512M`（Nginx）+ Livewire 规则 `max:500000` |
| Nginx 日志只有 favicon 404 | 缺图标，爬虫扫的 | 无害；放个 `public/favicon.ico` 即可消除 |
| 改了 .env 不生效 | 配置缓存 | `php8.4 artisan config:clear` |

## 十三、移动端收尾（APK）

APK 用 Capacitor 壳，靠 `server.url` 远程加载站点：
- 调试期：`server.url = http://192.168.1.x:8123`（同 WiFi，cleartext 仅调试）
- **上线后必须改**：`server.url = https://read.sxmnq.art`，关掉 `usesCleartextTraffic`，重新 `assembleDebug` 打包
- 因为 APK 直接读线上站点，前端/blade 改动硬刷新即见，无需每次重打 APK
