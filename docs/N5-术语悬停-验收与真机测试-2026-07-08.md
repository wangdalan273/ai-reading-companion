# N5 术语悬停 — 实现验收与真机测试指南

> 功能定位：阅读中遇到不懂的词/术语，**选中即就地解释**，不打断心流。
> 第一性原理：人脑在阅读时遇到生词，若跳出书本去查词典/百度，认知中断成本极高；把"解释"做成**选中词 → 气泡就地弹出**，把中断降到最低，是伴读感最轻量、最高频的增益。

---

## 一、改动清单

| 层 | 文件 | 改动 |
|---|---|---|
| 前端 | `resources/js/reader.js` | 选区浮条新增「📖 解释」按钮；新增 `_define()`（抓上下文→调接口→弹气泡）、`_selectionContext()`（取选中词所在段落作语境）、`_showDefPop()/_hideDefPop()/_esc()`（气泡渲染+转义防 XSS） |
| 前端 | `resources/css/app.css` | 新增 `.companion-def-pop` 解释气泡样式（暖琥珀主题、深浅色适配、关闭按钮、加载态） |
| 后端 | `app/Http/Controllers/CompanionController.php` | 新增 `define()`：校验 `term` → 复用 `LlmService::complete()` 生成结合语境的通俗解释（无密钥走 `mockDefine` 占位）→ 返回 `{ok, term, definition}`；新增 `mockDefine()` |
| 后端 | `app/Services/LlmService.php` | 新增 `isMockConfig()`，供端点判断离线演示模式 |
| 路由 | `routes/web.php` | 新增 `POST /api/companion/define`（auth 中间件） |

交互流：选中文本 → 浮条出现「📖 解释」→ 点击 → 在选区旁弹气泡（先显示"…解释中"）→ 接口返回后渲染解释 → 点 × 或页面其他地方关闭。

## 二、自测结果（已自测通过）

| 项 | 结果 |
|---|---|
| `php -l` 三文件（LlmService / CompanionController / routes） | ✅ 无语法错误 |
| `npm run build`（reader.js + app.css） | ✅ 构建通过（仅 chunk 体积提示，无害） |
| 真 HTTP `POST /api/companion/define`（JSON 姿势，与真机前端等价） | ✅ 返回 `{"ok":true,"term":"平行宇宙","definition":"…"}`，中文解释正确 |
| 离线演示 mock | ✅ 无密钥时返回占位说明，不报错 |

### 自测中发现并修复的问题
- **现象**：后端返回 `Malformed UTF-8 characters`（json_encode 失败）。
- **根因**：本轮最初用 **Edit 工具追加** 的中文片段，在本环境下字节损坏（静态中文字符串非法 UTF-8）。
- **修复**：改用 **Write 整文件重写** `CompanionController.php`（UTF-8 干净），重写后 JSON 姿势即刻返回 `ok:true`。
- **经验**：PHP 文件含中文的改动，优先用 Write 整文件重写，避免 Edit 追加中文导致字节损坏。

> 注：curl 表单编码（`--data-urlencode` 中文）在本环境会触发与上面无关的"中文解码怪癖"导致 Malformed，属 curl/Windows 测试姿势问题，**非应用缺陷**——真机前端发的是 JSON，已验证正常。

## 三、真机测试步骤

**前置**：后端已运行（`php artisan serve --host=127.0.0.1 --port=8123` 或你的部署）；已执行 `npm run build` 重新打包前端。

1. 浏览器打开入口 `http://127.0.0.1:8123/enter`（自动登录本机用户，进入书架）。
2. 书架上有一本 EPUB（没有就先上传一本），点击进入阅读器。
3. 在阅读正文里，用鼠标拖选 **一个词或短语**（例如「平行宇宙」「量子纠缠」），手机上长按选区。
4. 选区上方浮条会出现一排按钮，其中新增了 **「📖 解释」**。
5. 点击「📖 解释」——选区旁会弹出一张解释气泡卡片（先显示"…解释中"）。
6. 片刻后气泡显示该词 **结合上下文的通俗解释**（大白话 + 生活化例子）。
7. 点气泡右上角 **×**，或点击页面其他任意位置，气泡关闭。

**预期结果**：
- 在「⚙️ AI 设置」填了密钥 → 返回真实模型结合语境的精准解释。
- 未填密钥（离线演示）→ 返回通用占位说明（功能链路仍可走通，便于无 key 演示）。

## 四、常见问题与修复

| 现象 | 排查 / 修复 |
|---|---|
| 选中词没出现浮条 | 确认是在阅读器正文（epub.js iframe 内）选中，且已重新 `npm run build`；刷新页面清缓存 |
| 点「📖 解释」毫无反应 | 打开浏览器控制台 Network，看 `POST /api/companion/define` 是否 200；若 401 先 /enter 登录；若 500 看后端日志 |
| 解释内容不对题 / 很通用 | 离线演示模式会返回占位；去「AI 设置」填入可用密钥即变精准 |
| 气泡位置被遮挡 | 气泡已做视口边界翻转（上方空间不足自动落到下方），如仍异常请反馈具体浏览器 |

## 五、依赖与复用
- 复用 P11 的 `LlmService::complete()`（非流式三协议适配器），密钥不出服务端。
- 复用现有选区浮条机制（与「问 AI / 翻译」同一范式），零新增前端架构。
- 气泡样式沿用项目暖琥珀「温润书香」主题变量，深浅色自动适配。
