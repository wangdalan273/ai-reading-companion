## 一、从第一性原理做了什么

主动提取 > 被动阅读（测试效应）。P15 把"读懂"再推一层：

1. **苏格拉底模式**——AI 不直接给答案，只通过有层次的问题引导你自己悟出来。对话栏加开关，开启后所有提问都进入"只问不答"角色。
2. **自动测验**——据你选中的文字（或全书某章）自动出选择题，答完逐题给解析。这是把"读过的东西真正固化"的刻意练习。

两者都守住了项目铁律：**离线优先、不假死、Obsidian 友好**。

## 二、交付内容清单

### 苏格拉底模式
- `app/Http/Controllers/CompanionController.php`：`ask()` 的 `mode` 校验由 `in:devil` 放宽为 `in:devil,socratic`，`mode` 已原样透传给 `LlmService::stream` 并落库。
- `app/Services/LlmService.php`：`buildMessages()` 加 `socratic` 分支（只提问、不直答、温暖并肩散步语气）；`mockStream()` 加离线条（离线演示引导问题）。
- `resources/views/read.blade.php`：对话栏加「🧭 苏格拉底」开关，**与魔鬼代言人互斥**（开了苏格拉底自动关魔鬼，反之亦然）。
- `resources/views/layouts/app.blade.php`：`companionChat` store 加 `socratic` 状态 + `toggleSocratic()`，`_stream()` 发问时 `mode: devil ? 'devil' : (socratic ? 'socratic' : '')`。

### 自动测验
- 迁移 `2026_07_09_000006_create_quizzes_tables`：`quizzes`(user_id,book_id,chapter_title,source_type,source_ref,total) + `quiz_questions`(quiz_id,question,options_json,answer_index,explanation)。
- 模型 `Quiz` / `QuizQuestion`（hasMany + casts options_json→array）。
- `app/Http/Controllers/QuizController.php`：
  - `generate`：据 `selection`（选中文字）/ `book`（全书随机抽章）解析原文 → 真实模型出 JSON 题库（stem/options/answer/reason），**解析失败或 429 降级离线启发式** → 落库返回。
  - `show`：重新打开已存测验。
  - `submit`：判分 + 逐题正误 + 解析。
  - `export`：导出 Obsidian 友好 md；若配置了 vault 路径则直接写入 vault，否则返回浏览器下载。
- `app/Services/ExportService.php`：加 `toQuizMarkdown(Quiz)`（frontmatter + `source: "[[书名]]"` 双链 + `> [!quiz]`/`> [!question]` callout + 答案/解析）；对话导出标签已含 🧭苏格拉底。
- `resources/views/read.blade.php`：对话栏 header 加「📝 出测验」入口；底部加答题弹窗（来源选择 → 生成 → 单选 → 提交 → 得分+解析 → 导出/写入 Obsidian）。
- 路由：`book.quiz.generate`(POST) / `book.quiz.show`(GET) / `book.quiz.submit`(POST) / `book.quiz.export`(GET)，均 auth 网关。

## 四、真机验收步骤

1. 浏览器开 `http://127.0.0.1:8123/` → 登录 / `/enter` 免注册 → 打开一本书（`/read/{id}`）。
2. **苏格拉底模式**：在对话栏点「🧭 苏格拉底：关」→ 变「开」→ 输入一个问题或选中句子问 AI → 回答应**只提问、不给答案**；再点一下关闭，恢复正常解答。注意与「🎯 魔鬼代言人」互斥。
3. **自动测验（选中文字）**：在书中选中一段话 → 对话栏点「📝 出测验」→ 范围选"选中文字" → 「生成测验」→ 等生成 → 逐题点选项 → 「提交答案」→ 看得分 + 每题解析（绿对红错）。
4. **自动测验（全书）**：弹窗里范围选"全书抽取" → 生成 → 同样答题看解析。
5. **导出 Obsidian**：在测验弹窗点「📤 导出 / 写入 Obsidian」→
   - 若「AI 设置」里配了 Obsidian vault 路径 → 直接写入 vault（提示路径）；
   - 否则浏览器下载 `.md`，拖进 vault 即连网（`[[书名]]` 双链 + frontmatter + callout）。
6. **离线验证**：未填密钥时，苏格拉底给出离线引导问题、测验给出离线理解题（功能不假死）。

## 五、已知边界 / 后续可增强
- "当前章"精确出题暂用"全书随机抽章"兜底（`CompanionReader` 未直供当前章正文 API）；若要精确到当前可视章节，可在 reader.js 暴露 `getCurrentChapter()` 后由前端传 `chapter_title`，后端按标题精确匹配章节。
- 测验目前不自动关联划线笔记；后续可让"选中文字"出题时把该题 source_ref 指向该划线，便于回溯。
