# N7 人物关系图 + 时间线 · 真机测试文档（2026-07-09）

> 配套：`功能详细设计与排期-第一性原理-2026-07-08.md` Phase 2 之 N7。
> 复用范式：N3 概念图谱（GraphController 三方法 + 四字段落库 + 纯 canvas 力导向图）。

## 一、功能概述

在阅读页新增「👥 人物」入口，进入**人物关系图页**：从全书抽取人物、关系与关键事件，渲染成
- 交互式**力导向关系图**（节点=人物，按阵营 faction 着色；边=关系，带类型标签如 亲属/敌对/合作/师徒）
- 底部**时间线**（按序列出关键事件，点事件看剧情卡）

点人物 → 右侧详情面板显示：人物名、阵营标签、简介、出场章节、关联人物（关系列表）。
点时间轴事件 → 右上角卡片显示时间点、事件描述、涉及人物。

## 三、真机测试步骤

前置：服务已在 8123 运行（本次已重启）。浏览器打开 `http://127.0.0.1:8123/`（或 `/enter` 免注册进入）。

1. 打开一本**小说类**书（已导入的三国/小说类 epub 最佳）→ 进入阅读页。
2. 阅读页**顶部工具栏**（手机）或**右侧 AI 对话栏头部**（桌面）点「👥 人物」。
3. 在人物关系图页点右上角「🤖 生成 / 重新生成」。
4. 等待数十秒～数分钟（首次需抽取全书人物/关系/事件，后端已 `set_time_limit(0)` 防超时）。
5. 观察：力导向图浮现人物节点（按阵营着色）、关系边带类型标签；底部时间轴列出关键事件。
6. 交互验证：
   - 拖拽节点排版、滚轮缩放、拖背景平移；
   - 点人物 → 右侧详情（简介/阵营/出场章节/关联人物）；
   - 点时间轴事件 → 右上角剧情卡。

**预期**：
- 状态行显示「✅ 生成完成（人物 N · 关系 M · 事件 K）」。
- 关系图可交互、节点人名合理（非"方解/于中医"之类）。
- **下次进入该书「👥 人物」直接显示已生成图**（四字段落库持久化，无需重新生成）。

## 四、离线 vs 真实模型

- **未填密钥 / 密钥触发 429 限流**：自动降级为离线关键词共现图（重要概念关系，非精准人物），状态行提示
  「已用离线启发式生成；额度恢复后重填密钥并重新生成即可获得真实人物图」。
- **填入有效密钥且额度充足**：真实模型抽取精准人物——合并同一人不同称呼、标注阵营、关系类型、关键事件时间线。

## 五、已知限制

- 离线模式无人物简介、无时间线事件（前端提示「填入密钥后由真实模型生成」）。
- 非虚构书（`genre=nonfiction`）前端提示「本书疑似非虚构，人物关系可能不完整」。
- 人物共指/别名消歧依赖模型 prompt（真实模型已要求"合并同一人不同称呼，用最常用名"）。
- 移动端按鼠标交互设计，触屏拖拽未专门适配（与 N3 概念图谱一致，可接受）。
